# 開発振り返り - 2025 年 11 月 29 日

## 総評

今週は個人ブログの基盤改善に取り組みました。特に、記事ファイルの管理方法について見直しを行い、より効率的な運用ができる体制を整えました。

---

### できたこと・よかったこと

**1. コンテンツ管理の最適化**

- 記事ファイルの配置を `content/posts/` から `content/posts/active/` に変更
- `archive/` ディレクトリを作成し、古い記事の退避場所を確保
- `.gitignore` を適切に設定し、Git 管理の範囲を明確化

**2. ローカル環境の容量問題への対策**

- 記事とコードの分離について検討
- 画像ファイルの外部化（S3/Cloudinary）の方針を決定
- 今後のスケーラビリティを考慮した設計を検討

**3. Git 運用の理解が深まった**

- `git rm --cached` と `.gitignore` の違いを理解
- 既に追跡されているファイルの管理から外す方法を習得
- コミット履歴からのファイル復元方法を学習

---

### 学んだこと

**コンテンツ管理のベストプラクティス:**

1. **コードとコンテンツの分離**が重要
2. **画像ファイル**がローカル容量を圧迫する主要因
3. Headless CMS や S3 などの外部ストレージ活用が有効

**Git 管理の知見:**

- `.gitignore` に追加しても、既に追跡されているファイルは管理され続ける
- `git rm --cached` で追跡を解除する必要がある
- `pnpm-lock.yaml` の差分は依存関係の再計算による正常な変更であることが多い

---

### 課題

**1. 画像管理の外部化**

- 現在は `public/images` にローカル保存している
- S3 や Cloudinary への移行を検討中

**2. 記事執筆フローの効率化**

- 現在はローカルで MDX ファイルを作成 → Git にプッシュ
- より効率的なワークフローを模索したい

---

## 来週の目標

2. 記事執筆から公開までのフローを確立
