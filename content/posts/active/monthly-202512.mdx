# 月次振り返り - 2025 年 12 月

## はじめに

今月は、AI の出してきたコードや自分で書いたコードに対しての精査力、上司になぜこの設計で良いと判断したのかの設計力、提案力が先月よりもついたと感じる 1 ヶ月でした。
というのも社内アカウントで Claude Code が付与され、今まで以上に VibeCording に明け暮れた 1 ヶ月であるからです。
一方、API、UI/UX の設計を行う場面では、自分の設計ありきで AI に相談し、メリットデメリットを精査しないといけない場面があり、設計力もまだまだ知見をつけていく必要があります。さらにはその設計でなぜ良いと判断したのかの上司への説得性を持たせた提案の仕方、姿勢を学びました。

---

## VibeCording ドップリ

はじめにでも記載したように社内アカウントで Claude Code のアカウントを付与してもらったので、今まで以上に VibeCording な 1 ヶ月になりました。
Plan モードや、Edit モードなど使用の仕方は Copilot と変わらないですが、出力やファイル全体を調査する力は圧倒的に Claude Code の方が Copilot より高いと感じます。

今月は 8-9 割くらい API 側を書いていたので、設計部分や、より良いコードの書き方について対話していました。
また、その書いたコードを 1 度自分で確認した後に、Copilot の Rv で指摘があり、確かにとダブルパンチで考えさせられる機会が多かったです。
おかげで、コードの精査力が今まで以上に身につきました。

---

## 社内 LT をしました

私の今の PJ では週に 2 回進捗やインシデント共有の場があります。進行役の先輩がいつもはじめの 5 分間雑談をするのですが、テストについての本を読み、それを他人にアウトプットしたかったので、その雑談を私の LT 会にしてもらいました笑
読んだ内容を Notion にまとめてそれを共有する形で LT をしたのですが、思った以上に反響がよく、MJ やシニア BP さん、後輩からも良かったと反響をいただけました。

私は、性格上、締切駆動や、〇〇駆動みたいな学習方法が性に合っていると最近気づいたので、今後も記事でのアウトプットも含めてどんどん発信していこうと思いました。

また、社内での発信活動文化も作っていきたいなと感じました。

---

## 私生活の不安定

今月は特に金曜日や土日の PF が悪かったです。
もちろん日頃の疲れが溜まっているのだと思いますが、週末も安定して最低限の PF を出せるように、生活時間をなるべく一定にする、睡眠時間を最低 8h は取れるように工夫するなど、来年は今年以上に PF を安定させるように意識していきたいです。

一方で、去年や今年の 10 月以前と比較すると、出勤時刻と退勤時刻、トレーニングの開始、終了時刻は一定していたので、比較的月-木の PF は以前より良くなっています。
これも継続していきたいです。

## 提案力大事！

業務で UI/UX の改修があり、社内のドメインに関わる箇所で背景がわからない箇所だったので、上司に確認しました。しかし、上司も把握していない、昔在籍していた人が設計したような箇所でした。
私は上司に自分の提案や意思を提示せず、「ここも他と統制するように修正する形で良いですか？」と一方的な質問の仕方をしていました。
上司からは「なぜその設計方針にしたのか、ユースケースを考慮してメリット、デメリットを自分なりに精査した上で再度提案して」とアドバイスを受け、再度提案し、「その提案いいね！」と言っていただきました。
このようなきっかけを与えてくれた上司に感謝です！
